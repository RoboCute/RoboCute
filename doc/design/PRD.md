# RoboCute产品需求文档

## 目标工作流

考虑到行业内大部分的算法开发生态都在python环境下，RoboCute也会坚持python-first，让python环境的启动作为首选。

### 物理模拟工作流

- 安装: `pip install`
- 核心工作流产品类似ComfyUI
  - 入口`python main.py`，传入适当命令行开启窗口和服务器
  - python端实现：
    - 定义节点，输入/输出格式，方法(Node Description)
    - 存储场景和节点定义，注册对应的方法调用入口和参数接口
    - 开启窗口
    - 开启控制server
  - cpp端实现：
    - 窗口开启后，打开编辑场景，监听控制server
    - 根据定义的节点连接节点图 (Editor Node Graph)
    - 根据场景对场景进行编辑和参数给定
    - 如果server信号正常，则可以执行节点图
    - 启动执行节点图，并将控制参数传给server
  - python端实现
    - 接受cpp窗口端的执行图，同步场景参数
    - 按照控制参数构建运行时执行图(Runtime Graph)
    - 执行Runtime Graph，将结果返回给编辑器
  - cpp端：
    - 查询执行结果
    - 可视化（重播，可视化过程参数，细节参考，debug界面等）
    - 编辑调整
    - 重新运行
- 拓展开发：
  - 用户可以使用python自定义场景中的算法策略，如寻路，操作，AI等，只要将结果最终封装为一个标准节点注册进入节点图就可以在编辑器中编辑使用
  - 数据驱动交互：用户可以从标准场景节点中拓展更多需要的数据接口（比如声音，碰撞等）封装传感器来提高机器人的仿真实现
  - 用户可以拓展编辑器功能，提供更多有用的可视化信息
- 部署：
  - 在仿真测试完成后，需要能对接实机标准进行部署

### AIGC工作流

AIGC工作流和物理模拟工作流不同的地方在于，并不存在一个初始场景，而是完全从节点图开始
